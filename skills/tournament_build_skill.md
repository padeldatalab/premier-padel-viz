# Skill: Premier Padel Tournament Build

## Purpose
Given a Premier Padel / FIP draw PDF and basic tournament metadata, produce:
1. An individual tournament JSON file conforming to the shared schema
2. A self-contained tournament HTML file (Summary + Performance + Draw tabs) that fetches from the JSON

Both files are ready to push directly into the `padeldatalab/premier-padel-viz` GitHub Pages repo.

## Step 1 — Read the PDF carefully

### PDF structure
Premier Padel draw PDFs have two pages: **Top Half** and **Bottom Half**.
Each page shows columns left to right: R64 (or R32 for smaller draws) → R32 → R16 → QF → SF → Final.

### Critical reading rules
- **Read match opponents from the draw column, not from seedings alone.** Inferring opponents from seedings leads to errors — always trace the actual draw lines.
- **Scores are shown next to the winner's name** in each subsequent round column.
- **BYE** = team advances without playing. The advancing team's `score` is `null` and `walkover` is `true`.
- **Retirements**: noted as "RET" in the score. Set `retired_team_id` to the team that retired. Include sets played up to retirement.
- **Walkovers**: team wins without playing at all (opponent withdrew before match). `walkover: true`, `score: null`.
- **Qualifier teams** are marked "Q" on the draw. Entry type = `"Q"`.
- **Wildcards** are marked "WC". Entry type = `"WC"`.
- **Seeded teams** are numbered [1]–[16] (or [1]–[8] for P2). Entry type = `"S"`.
- **All other teams** are direct acceptances. Entry type = `"DA"`.

### Score format in JSON
Each set is an object:
```json
{ "set": 1, "t1": 6, "t2": 3 }
```
Tiebreak scores add a `"tiebreak"` field for the loser's tiebreak points:
```json
{ "set": 2, "t1": 7, "t2": 6, "tiebreak": 4 }
```
`t1` is always team1 (top team in the match card), `t2` is always team2.

### After reading
- Count total matches extracted. Expected: 63 for 64-draw, 31 for 32-draw.
- Verify every seeded team appears exactly once in the draw.
- Verify the winner chain: the team winning each match appears as a participant in the next round match for that section.
- Flag any discrepancies before proceeding.

## Step 2 — File naming conventions

### JSON filename
```
YYYY_MM_DD_TournamentName_gender.json
```
Examples:
- `2026_05_31_Italy_major_men.json`
- `2026_06_14_Valladolid_P2_men.json`
- `2026_06_21_Valencia_P1_men.json`

Use the **start date** of the tournament, underscores throughout, no spaces.

### HTML filename
Same as JSON but `.html`:
- `2026_05_31_Italy_major_men.html`

### tournament_id
Lowercase, underscores, no spaces:
```
{tournamentname}_{tier}_{year}
```
Examples: `italy_major_2026`, `valladolid_p2_2026`, `valencia_p1_2026`

### match_id
```
{short_tid}_{round}_{half}_{n}
```
- `short_tid`: first word of location + tier abbreviation + year e.g. `italy26`, `valencia26`, `valladolid26`
- `round`: r64, r32, r16, qf, sf, final (lowercase)
- `half`: t (top) or b (bottom)
- `n`: match number within round and half (1-indexed)

Examples: `italy26_r64_t01`, `italy26_qf_b02`, `italy26_final`

## Step 3 — Points structures per tier

### Major (64-draw)
```json
{
  "R64":   { "per_player": 35,   "team_total": 70   },
  "R32":   { "per_player": 90,   "team_total": 180  },
  "R16":   { "per_player": 180,  "team_total": 360  },
  "QF":    { "per_player": 360,  "team_total": 720  },
  "SF":    { "per_player": 720,  "team_total": 1440 },
  "Final": { "per_player": 1200, "team_total": 2400 },
  "W":     { "per_player": 2000, "team_total": 4000 }
}
```

### P1 (48 or 32-draw)
```json
{
  "R32":   { "per_player": 60,   "team_total": 120  },
  "R16":   { "per_player": 120,  "team_total": 240  },
  "QF":    { "per_player": 240,  "team_total": 480  },
  "SF":    { "per_player": 480,  "team_total": 960  },
  "Final": { "per_player": 800,  "team_total": 1600 },
  "W":     { "per_player": 1400, "team_total": 2800 }
}
```

### P2 (32-draw)
```json
{
  "R32":   { "per_player": 30,   "team_total": 60   },
  "R16":   { "per_player": 60,   "team_total": 120  },
  "QF":    { "per_player": 120,  "team_total": 240  },
  "SF":    { "per_player": 240,  "team_total": 480  },
  "Final": { "per_player": 400,  "team_total": 800  },
  "W":     { "per_player": 700,  "team_total": 1400 }
}
```

**Note:** Points are per-player in the source data. Team total = per_player × 2.
Finalist teams both receive the `Final` allocation. Only the winner receives `W`.

## Step 4 — JSON schema

Build the JSON with these five top-level arrays. Every field matters — do not omit or abbreviate.

### tournaments
One object per file:
```json
{
  "tournament_id": "italy_major_2026",
  "name": "Italy Major",
  "edition": 2026,
  "tier": "Major",
  "location": { "city": "Rome", "country": "Italy", "country_code": "ITA" },
  "dates": { "start": "2026-05-31", "end": "2026-06-07" },
  "prize_fund_eur": 1044849,
  "draw_size": 64,
  "gender": "men",
  "rounds": ["R64", "R32", "R16", "QF", "SF", "Final"],
  "points_structure": { ... per tier above ... },
  "prize_structure_eur": { ... fill from PDF or set all to 0 if unknown ... }
}
```

### teams
One object per unique team. Include ALL teams in the draw (seeded, unseeded, qualifiers, wildcards). Do not include BYE as a team.
```json
{
  "team_id": "tapia_coello",
  "players": [
    { "surname": "Tapia",  "first_name": "Agustin", "nationality": "ARG" },
    { "surname": "Coello", "first_name": "Arturo",  "nationality": "ESP" }
  ]
}
```
- `team_id`: `{surname1}_{surname2}` lowercase, underscores, no accents in the ID
- Use full legal surnames (not abbreviations) in the `surname` field
- Nationality = 3-letter IOC country code

### tournament_entries
One object per team entry. Seeded teams only have seed numbers; all others have `null`.
```json
{
  "tournament_id": "italy_major_2026",
  "team_id": "tapia_coello",
  "seed": 1,
  "entry_type": "S",
  "fip_points_pre_tournament": 42218,
  "draw_half": "top",
  "draw_position": 1
}
```
- `entry_type`: `"S"` (seeded), `"DA"` (direct acceptance), `"Q"` (qualifier), `"WC"` (wildcard), `"LL"` (lucky loser)
- `fip_points_pre_tournament`: from the seedings table on the PDF for seeded teams; `null` for others
- `draw_position`: 1–64 (or 1–32) numbered top to bottom through the full draw

### matches
One object per match. Include BYE matches (walkover=true, score=null).
```json
{
  "match_id": "italy26_r64_t01",
  "tournament_id": "italy_major_2026",
  "round": "R64",
  "draw_half": "top",
  "match_number": 1,
  "team1_id": "tapia_coello",
  "team2_id": "bye",
  "winner_id": "tapia_coello",
  "score": null,
  "walkover": true,
  "retired_team_id": null
}
```
- `draw_half`: `"top"`, `"bottom"`, or `null` for the Final
- `match_number`: 1-indexed within each round+half combination
- `score`: array of set objects, or `null` for walkovers/byes
- For retirements: include sets played, set `retired_team_id` to the team that retired

### tournament_results
One object per team that participated. Round reached = the round they were **eliminated in** (or `"W"` for the winner).
```json
{
  "tournament_id": "italy_major_2026",
  "team_id": "tapia_coello",
  "seed": 1,
  "round_reached": "W",
  "points_earned_per_player": 2000,
  "points_earned_team": 4000
}
```
- Teams eliminated in R64 get `"round_reached": "R64"` (they played and lost in R64)
- Teams with a BYE who then lose in R32 get `"round_reached": "R32"`
- Both finalists get `"Final"` — only the winner gets `"W"`
- Include ALL teams who played at least one match (including BYE winners who then lost)

## Step 5 — JSON validation checklist

Before outputting the JSON, verify:
- [ ] Match count: 63 matches for 64-draw (including byes), 31 for 32-draw
- [ ] Every match has a `winner_id` that matches either `team1_id` or `team2_id`
- [ ] No team appears twice in the same round on the same half
- [ ] Winner of each match appears in the correct next-round match
- [ ] All seeded teams appear in `tournament_entries`
- [ ] `tournament_results` has an entry for every team in the draw (excluding BYE)
- [ ] Points in results match the `points_structure` for the round reached
- [ ] Finalist has `"round_reached": "Final"`, winner has `"round_reached": "W"`
- [ ] No `team_id` in matches references a team not in the `teams` array (except `"bye"`)

## Step 6 — HTML file

### Template source
The HTML template is `2026_05_31_Italy_major_men.html` from the repo's `tournaments/` folder.
It contains three tabs: **Summary**, **Performance**, **Draw**.
All rendering is driven by the JSON — no hardcoded match data in the HTML.

### What to change for a new tournament
1. `<title>` tag — update to new tournament name
2. `JSON_PATH` constant — update filename:
   ```javascript
   return up + '/data/tournaments/YYYY_MM_DD_Name_gender.json';
   ```
3. `#tourney-title` content — update tournament name
4. `#tourney-meta` content — update city, dates, gender
5. `.tier-pill` content — update tier (Major / P1 / P2)
6. The retirement notice in the summary HTML — update or remove:
   ```html
   <div class="s-notice">...</div>
   ```
   Remove entirely if no retirements. Update text if there were retirements.

### What does NOT change
- All CSS styles
- All three build functions (`buildSummary`, `buildChart`, `buildBracket`)
- `transformJSON()` function — it reads the schema generically
- Nav tab logic
- Zoom controls

### Performance chart — expected round logic
The `expIdx()` function inside `buildChart` determines what round each seed is expected to reach:
```javascript
function expIdx(seed) {
  if (!seed)    return 2;  // unseeded → R16 band
  if (seed<=2)  return 5;  // seeds 1-2 → Final (winning = overperform)
  if (seed<=4)  return 4;  // seeds 3-4 → SF
  if (seed<=8)  return 3;  // seeds 5-8 → QF
  if (seed<=12) return 2;  // seeds 9-12 → R16
  return 1;                // seeds 13-16 → R32
}
```
This works for 64-draw Major. For P1/P2 (32-draw, 8 seeds), adjust as:
```javascript
if (seed<=2)  return 5;  // seeds 1-2 → Final
if (seed<=4)  return 4;  // seeds 3-4 → SF
return 3;                // seeds 5-8 → QF
```

## Reference: draw sizes and seedings by tier

| Tier   | Draw size | Seeds | Rounds                        |
|--------|-----------|-------|-------------------------------|
| Major  | 64        | 16    | R64, R32, R16, QF, SF, Final  |
| P1     | 48 or 32  | 16    | R32, R16, QF, SF, Final       |
| P2     | 32        | 8     | R32, R16, QF, SF, Final       |

P1 48-draw: top 16 seeds get a bye into R32, so R32 has 16 matches (8 byes + 8 played).

## Common errors to avoid

1. **Misreading bottom-half draw order** — the bottom half reads top-to-bottom on the PDF page but is the mirror of the top half in the bracket. Match numbers in the bottom half start from 1 again within each round.

2. **Confusing score direction** — scores are always shown next to the winner. The first number is the winner's games, not always team1's. Always check which team won before assigning t1/t2 scores.

3. **Duplicate yanguas_stupaczuk in results** — if a team appears twice in tournament_results (once for the round they lost, once for an earlier round), keep only the highest round reached.

4. **Finalist points** — both finalists get `Final` points (per_player × 2). The winner additionally gets `W` points, not `Final` points. They are mutually exclusive.

5. **BYE team_id** — use the string `"bye"` (lowercase) as the `team_id` for bye opponents. Do not add BYE to the `teams` array.

6. **Tiebreak field** — `tiebreak` stores the **loser's** tiebreak points, not the winner's. A 7-6(4) set means winner got 7, loser got 6, loser's tiebreak score was 4.

7. **draw_half for Final** — set `"draw_half": null` for the Final match. It belongs to neither half.
