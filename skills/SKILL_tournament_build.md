# Skill: Premier Padel Tournament Build

## Purpose
Given a Premier Padel / FIP draw PDF, produce a single tournament JSON file conforming
to the shared schema. The JSON is all that changes between tournaments — the HTML is a
generic template that reads any JSON at runtime.

**Output:** one JSON file, ready to push to the repo.  
**No HTML generation required.** The template (`tournaments/_template.html`) handles all
rendering automatically from the JSON.

## Step 1 — Read the PDF carefully

### PDF structure
Two pages: **Top Half** and **Bottom Half**.  
Each page shows columns left to right: first round → R32 → R16 → QF → SF → Final.

### Critical reading rules
- **Trace actual draw lines** — never infer opponents from seedings alone.
- **Scores appear next to the winner's name** in each subsequent round column.
- **BYE**: team advances without playing → `score: null`, `walkover: true`.
- **Retirement** (RET in score): include sets played up to retirement, set `retired_team_id`.
- **Walkover**: opponent withdrew before the match → `walkover: true`, `score: null`.
- Entry types: `"S"` seeded `[n]`, `"Q"` qualifier, `"WC"` wildcard, `"LL"` lucky loser, `"DA"` everyone else.

### Score format
```json
{ "set": 1, "t1": 6, "t2": 3 }
```
Tiebreak: `tiebreak` field holds the **loser's** tiebreak points.
```json
{ "set": 2, "t1": 7, "t2": 6, "tiebreak": 4 }
```
`t1` = top team in the match card, `t2` = bottom team. Always assign from team1's perspective
regardless of who won.

### After reading — verify before encoding
- Count expected matches: 63 for 64-draw, 61 for 32-draw (includes byes), 47 for 48-draw.
- Confirm every seeded team appears exactly once.
- Trace the winner chain: winner of each match must appear in the correct next-round slot.
- Flag any discrepancies before writing JSON.

## Step 2 — File naming

### JSON filename
```
YYYY_MM_DD_TournamentName_gender.json
```
Use start date, underscores, no spaces, no accents.  
Examples: `2026_02_07_Riyadh_P1_men.json` · `2026_05_31_Italy_major_men.json`

### tournament_id
```
{city/name}_{tier}_{year}    (all lowercase, underscores)
```
Examples: `riyadh_p1_2026` · `italy_major_2026` · `valladolid_p2_2026`

### match_id
```
{short_tid}_{round}_{half}{n}
```
- `short_tid`: first word of city + tier abbreviation + 2-digit year → `riyadh26`, `italy26`
- `round`: `r64`, `r32`, `r16`, `qf`, `sf`, `final`
- `half`: `_t` (top) or `_b` (bottom); omit for Final
- `n`: zero-padded 2-digit index within round+half

Examples: `riyadh26_r32_t01` · `riyadh26_qf_b02` · `riyadh26_final`

## Step 3 — Points structures

### Major (64-draw)
| Round | per_player | team_total |
|-------|-----------|-----------|
| R64   | 35  | 70   |
| R32   | 90  | 180  |
| R16   | 180 | 360  |
| QF    | 360 | 720  |
| SF    | 720 | 1440 |
| Final | 1200| 2400 |
| W     | 2000| 4000 |

### P1 (32 or 48-draw)
| Round | per_player | team_total |
|-------|-----------|-----------|
| R32   | 60  | 120  |
| R16   | 120 | 240  |
| QF    | 240 | 480  |
| SF    | 480 | 960  |
| Final | 800 | 1600 |
| W     | 1400| 2800 |

### P2 (32-draw)
| Round | per_player | team_total |
|-------|-----------|-----------|
| R32   | 30  | 60   |
| R16   | 60  | 120  |
| QF    | 120 | 240  |
| SF    | 240 | 480  |
| Final | 400 | 800  |
| W     | 700 | 1400 |

**Rules:** Both finalists get `Final` points. Winner gets `W` points instead, not in addition.  
team_total = per_player × 2.

## Step 4 — JSON schema

Five top-level arrays. For full working examples of every field, see the reference JSON:  
`data/tournaments/2026_02_07_Riyadh_P1_men.json`

### tournaments (1 object)
```json
{
  "tournament_id": "riyadh_p1_2026",
  "name": "Riyadh Season P1",
  "edition": 2026,
  "tier": "P1",
  "location": { "city": "Riyadh", "country": "Saudi Arabia", "country_code": "KSA" },
  "dates": { "start": "2026-02-07", "end": "2026-02-14" },
  "prize_fund_eur": 479068,
  "draw_size": 32,
  "gender": "men",
  "rounds": ["R32", "R16", "QF", "SF", "Final"],
  "points_structure": { ... from Step 3 ... },
  "prize_structure_eur": { "R32": 0, "R16": 0, "QF": 0, "SF": 0, "Final": 0, "W": 0 }
}
```
`rounds` must not include `"W"` — that is only used in `points_structure` and `tournament_results`.

### teams (1 per unique team)
```json
{
  "team_id": "coello_tapia",
  "players": [
    { "surname": "Coello", "first_name": "Arturo",  "nationality": "ESP" },
    { "surname": "Tapia",  "first_name": "Agustin", "nationality": "ARG" }
  ]
}
```
- `team_id`: `surname1_surname2`, lowercase, no accents, no spaces
- Do NOT add a `bye` team object
- Nationality = 3-letter IOC country code

### tournament_entries (1 per team)
```json
{
  "tournament_id": "riyadh_p1_2026",
  "team_id": "coello_tapia",
  "seed": 1,
  "entry_type": "S",
  "fip_points_pre_tournament": 39600,
  "draw_half": "top",
  "draw_position": 1
}
```
- `seed`: integer for seeded teams, `null` for all others
- `fip_points_pre_tournament`: from the seedings table on the PDF; `null` for unseeded
- `draw_position`: 1-indexed top-to-bottom through the full draw

### matches (1 per match, including byes)
```json
{
  "match_id": "riyadh26_r32_t01",
  "tournament_id": "riyadh_p1_2026",
  "round": "R32",
  "draw_half": "top",
  "match_number": 1,
  "team1_id": "coello_tapia",
  "team2_id": "bye",
  "winner_id": "coello_tapia",
  "score": null,
  "walkover": true,
  "retired_team_id": null
}
```
- `draw_half`: `"top"`, `"bottom"`, or `null` for the Final only
- `team2_id`: use `"bye"` (lowercase string) for bye slots
- `score`: `null` for byes/walkovers; array of set objects for played matches

### tournament_results (1 per team, excluding bye)
```json
{
  "tournament_id": "riyadh_p1_2026",
  "team_id": "coello_tapia",
  "seed": 1,
  "round_reached": "W",
  "points_earned_per_player": 1400,
  "points_earned_team": 2800
}
```
- `round_reached`: the round a team **last participated in** — win or loss
  - Lost in R32 → `"R32"`
  - Had a bye in R32, lost in R16 → `"R16"`
  - Reached the Final and lost → `"Final"`
  - Won the tournament → `"W"`
- Each team appears exactly once — keep only the highest round if duplicates arise

## Step 5 — Validation (run before delivering)

```python
import json
from collections import Counter

with open('YYYY_MM_DD_Name_gender.json') as f:
    d = json.load(f)

matches  = d['matches']
teams    = {t['team_id'] for t in d['teams']}
results  = d['tournament_results']

# Match count
print(len(matches))  # 61 for 32-draw, 63 for 64-draw

# All winners are team1 or team2
assert all(m['winner_id'] in (m['team1_id'], m['team2_id']) for m in matches)

# No unknown team refs (except 'bye')
for m in matches:
    for f in ['team1_id','team2_id','winner_id']:
        assert m[f] in teams or m[f] == 'bye', f"Unknown: {m[f]}"

# No duplicate teams in results
dupes = {k:v for k,v in Counter(r['team_id'] for r in results).items() if v > 1}
assert not dupes, f"Duplicates: {dupes}"

# Exactly one winner, two finalists
assert sum(1 for r in results if r['round_reached'] == 'W') == 1
assert sum(1 for r in results if r['round_reached'] == 'Final') == 2

print("All checks passed")
```

## Common errors to avoid

1. **Score direction** — scores sit next to the winner's name. The first number shown is the winner's games. Always establish who won before assigning `t1`/`t2` values.

2. **Tiebreak field is the loser's points** — `7-6(4)` means winner 7, loser 6, loser's tiebreak score 4.

3. **Bottom-half match numbering** — match numbers restart at 1 for each round+half. A bottom-half R16 has match_number 1–8, not 9–16.

4. **Finalist points are mutually exclusive with winner points** — finalist gets `Final` allocation. Winner gets `W` allocation only, not `Final` + `W`.

5. **Duplicate results** — if a team appears twice in `tournament_results`, keep only the entry with the highest `round_reached`. The Python validation above will catch this.

6. **draw_half for Final** — always `null`. The Final belongs to neither half.

7. **BYE is not a team** — `"bye"` is only used as a `team_id` string in match objects. No entry in the `teams` array.
